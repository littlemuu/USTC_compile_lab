## 本次更新说明

本次在 `task1_length_unit.cpp` 中补全了表达式计算和结果输出部分，主要修改如下：

1. 增加了 `<iostream>` 和 `<sstream>` 头文件。
   - `<iostream>` 用于 `main` 函数中读取输入和输出结果；
   - `<sstream>` 用于把 `double` 数值和单位字符串拼接成最终输出。

2. 增加了 `lengthUnitToString` 函数。

```cpp
std::string lengthUnitToString(LengthUnit unit) {
    if(unit==LengthUnit::Millimeter) return "mm";
    if(unit==LengthUnit::Centimeter) return "cm";
    if(unit==LengthUnit::Decimeter) return "dm";
    if(unit==LengthUnit::Meter) return "m";
    return "";
}
```

这个函数负责把枚举类型的单位转换成字符串，例如把 `LengthUnit::Centimeter` 转换成 `"cm"`。

3. 补全了 `evaluateLengthExpression` 的计算过程。
   - 使用 `numberStack` 保存数字；
   - 使用 `operatorStack` 保存运算符；
   - 遇到 `+` 或 `-` 时，先计算栈中已有的同级运算；
   - 遇到 `(` 时压入运算符栈；
   - 遇到 `)` 时一直计算到左括号；
   - 遍历结束后，再把运算符栈中剩余的运算全部执行完。

4. 修复了右括号处理后的逻辑。

右括号分支执行完：

```cpp
operatorStack.pop_back();
```

表示弹出左括号，之后应该继续处理下一个 token，不能让后面的普通数字分支和右括号分支混在一起。

5. 补全了 `formatLengthResult`。

```cpp
std::string formatLengthResult(const LengthResult& result) {
    if(!result.ok) return result.message;
    std::ostringstream oss;
    oss<<result.value<<lengthUnitToString(result.unit);
    return oss.str();
}
```

这里没有直接使用 `std::to_string`，因为 `std::to_string` 会输出较多小数位，例如 `16.000000cm`。使用 `std::ostringstream` 输出更自然。

6. 增加了 `main` 函数。

```cpp
int main(){
    std::string expression;
    std::getline(std::cin,expression);
    LengthResult result=evaluateLengthExpression(expression);
    std::cout<<formatLengthResult(result)<<std::endl;
    return 0;
}
```

程序运行时从标准输入读取一整行表达式，然后输出计算结果。

本次验证过的样例包括：


## Task1：长度单位表达式的解析与计算

本任务的目标是对带有长度单位的表达式进行解析、校验、单位统一和计算。例如：

```cpp
1m + 20cm - 3mm
```

程序需要先把表达式拆成一个个 token，再解析出数值和单位，最后统一到表达式中出现的最小单位后进行加减运算。

## 1. 定义长度单位和返回结果结构

首先定义 `LengthUnit` 枚举类型，用来表示不同长度单位：

```cpp
enum class LengthUnit {
    None,
    Millimeter,
    Centimeter,
    Decimeter,
    Meter
};
```

其中 `None` 表示没有单位，后面的枚举值分别表示 `mm`、`cm`、`dm` 和 `m`。

同时定义 `LengthValue` 保存一个长度值的信息：

```cpp
struct LengthValue {
    double value;
    LengthUnit unit;
    bool hasUnit;
};
```

这里：

- `value` 保存数值；
- `unit` 保存单位；
- `hasUnit` 表示这个数字是否带单位。

`LengthResult` 用来表示整个表达式的计算结果，包括是否成功、结果数值、结果单位和错误信息。

## 2. 判断字符是否为运算符

为了分词方便，先写了 `isOperator` 函数：

```cpp
bool isOperator(char ch){
    if(ch=='+'||ch=='-'||ch=='('||ch==')') return true;
    return false;
}
```

这个函数用于判断当前字符是不是 `+`、`-`、`(`、`)` 之一。

## 3. 表达式分词

`tokenizeLengthExpression` 函数负责把原始字符串拆成若干 token。

例如：

```cpp
1m + (20cm - 3mm)
```

会被拆成：

```cpp
["1m", "+", "(", "20cm", "-", "3mm", ")"]
```

实现时使用下标 `i` 遍历字符串：

- 如果遇到空白字符，直接跳过；
- 如果遇到运算符，就把它作为单独的 token；
- 如果遇到数字或单位，就继续向后读取，直到遇到空白字符或运算符为止。

这样可以把 `12.5cm` 这种“数字 + 单位”的整体作为一个 token 保存下来。

## 4. 解析长度单位

`parseLengthUnit` 函数负责把字符串形式的单位转换成枚举值：

```cpp
if(unitText=="mm") return LengthUnit::Millimeter;
if(unitText=="cm") return LengthUnit::Centimeter;
if(unitText=="dm") return LengthUnit::Decimeter;
if(unitText=="m") return LengthUnit::Meter;
return LengthUnit::None;
```

如果没有匹配到合法单位，就返回 `LengthUnit::None`。

## 5. 解析长度字面量

`parseLengthLiteral` 函数负责解析类似下面这样的 token：

```cpp
12.5cm
3m
100
```

这里使用 `std::stod` 把字符串开头的数字部分转换成 `double`：

```cpp
std::size_t pos = 0;
double value = std::stod(literalText, &pos);
```

`pos` 会记录数字部分结束的位置，因此可以用：

```cpp
std::string unitText = literalText.substr(pos);
```

取出后面的单位部分。

如果 `unitText` 是空字符串，说明这个 token 是普通数字；否则就继续调用 `parseLengthUnit` 解析单位。

## 6. 校验操作数是否合法

`validateLengthOperands` 用于拒绝“带单位数字”和“普通数字”混合出现的情况。

例如：

```cpp
1m + 2cm
```

是合法的，因为两个数字都带单位。

但是：

```cpp
1m + 2
```

是不合法的，因为一个数字带单位，另一个数字没有单位。

实现时使用两个布尔变量：

```cpp
bool hasUnitOperand = false;
bool hasPlainNumber = false;
```

遍历所有 `LengthValue`：

- 如果 `value.hasUnit` 为真，说明遇到了带单位数字；
- 否则说明遇到了普通数字；
- 如果两种情况都出现了，就返回 `false`。

## 7. 查找表达式中的最小单位

为了避免小数误差和方便计算，需要把表达式中的所有长度统一成最小单位。

例如：

```cpp
1m + 2cm + 3mm
```

其中最小单位是 `mm`，所以最终应该统一成毫米计算。

`findSmallestUnit` 的输入是：

```cpp
const std::vector<LengthValue>& values
```

也就是表达式中解析出来的所有长度值。

由于枚举值的顺序是：

```cpp
None = 0
Millimeter = 1
Centimeter = 2
Decimeter = 3
Meter = 4
```

所以可以通过比较枚举值对应的整数大小来找最小单位。

## 8. 单位转换

`convertToUnit` 函数用于把一个数值从原单位转换到目标单位。

当前思路是：如果从较大单位转换到较小单位，就每下降一级乘以 10。

例如：

```cpp
1m = 10dm = 100cm = 1000mm
```

所以把 `1m` 转成 `mm`，需要乘以 `10 * 10 * 10`。

## 9. 表达式求值

`evaluateLengthExpression` 是整个 Task1 的核心函数，主要流程如下：

1. 调用 `tokenizeLengthExpression` 得到所有 token；
2. 遍历 token，把非运算符 token 解析成 `LengthValue`；
3. 调用 `validateLengthOperands` 检查是否混合了带单位数字和普通数字；
4. 调用 `findSmallestUnit` 找到最小单位；
5. 遍历所有长度值，调用 `convertToUnit` 统一单位；
6. 使用数字栈和运算符栈完成加减法和括号表达式的计算。

计算时可以使用两个栈：

```cpp
std::vector<double> numberStack;
std::vector<char> operatorStack;
```

其中：

- `numberStack` 保存已经统一单位后的数值；
- `operatorStack` 保存 `+`、`-`、`(` 等运算符；
- 遇到数字时压入数字栈；
- 遇到运算符时根据优先级进行计算；
- 遇到右括号时，一直计算到左括号为止。

因为本题只涉及 `+` 和 `-`，所以加减法优先级相同，主要需要额外处理括号。

## 10. 格式化输出

最后通过 `formatLengthResult` 把 `LengthResult` 转换成字符串输出。

如果 `result.ok` 为 `false`，应输出错误信息；如果计算成功，则输出结果数值和对应单位。

例如：

```cpp
123mm
10cm
```

## 小结

Task1 的整体实现可以分为四个阶段：

1. 字符串分词；
2. 数字和单位解析；
3. 合法性检查与单位统一；
4. 根据 token 顺序进行表达式计算。

其中最关键的是：解析出来的 `tokens` 用来保留表达式的结构，解析出来的 `values` 用来保存所有数值及单位。计算时需要同时利用这两部分信息：按 `tokens` 的顺序判断当前是数字、运算符还是括号，遇到数字时再从 `values` 中取出已经统一单位后的数值。
