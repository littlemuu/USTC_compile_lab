**实验：基于 Flex 与 Bison 的语义翻译系统设计**

**一、实验目的**

1.  掌握使用 Flex 和 Bison 进行词法分析与语法分析的基本方法。

2.  理解**语义翻译**的概念，能够在语法规约过程中执行有意义的语义动作，包括：

    - 类型检查与单位统一（长度单位对齐到最小单位）

    - 数值进制转换（二进制字面量转十进制）

    - 生成中间表示（为四则运算生成基于栈的虚拟机指令代码）

## 二、实验内容

支持以下三种语义翻译子功能，可分别实现：

### 任务1：长度单位自动对齐与计算

- **输入**：包含长度单位的算术表达式，单位支持 mm, cm, dm, m；运算符支持 +、-；括号。

- **语义翻译规则**：

  - 自动识别表达式**中出现的最小长度单位**，将所有数值统一转换为该单位后再进行加减运算。

  - 若表达式中有纯数字（无单位），则不能与带单位的量混合（报语义错误）。

  - 输出结果时带上对应的单位名称。

- **示例**：

  - 输入 1m + 50cm → 输出 150.00 cm（最小单位是 cm）

  - 输入 2dm + 5mm → 输出 205.00 mm（最小单位是 mm）

### 任务2：二进制字面量转十进制

- **输入**：以 0b 或 0B 开头的二进制数字串，例如 0b1010，0B11001。

- **语义翻译规则**：

  - 解析该二进制字面量，将其转换为十进制整数值。

  - 转换后的数值可作为普通数字参与后续的四则运算（如 0b1010 + 5）。

- **示例**：

  - 输入 0b1010 → 输出 10

  - 输入 0b1010 + 3 → 输出 13

### 任务3：四则运算的栈代码生成

- **输入**：包含 +、-、\*、/ 和括号的整型或浮点型算术表达式。

- **语义翻译规则**：

  - 不直接计算数值，而是生成基于栈的虚拟机指令序列（例如 PUSH、ADD、SUB、MUL、DIV 等）。

  - 指令生成采用**逆波兰表示法**的隐式栈模型：遇到操作数则生成 PUSH 指令；遇到运算符则生成对应的运算指令。

  - 输出生成的指令列表。

- **示例**：

  - 输入 3 + 4 \* 2 → 输出指令序列：

> PUSH 3
>
> PUSH 4
>
> PUSH 2
>
> MUL
>
> ADD

- 输入 (1 + 2) \* 3 → 输出指令序列：

> PUSH 1
>
> PUSH 2
>
> ADD
>
> PUSH 3
>
> MUL
